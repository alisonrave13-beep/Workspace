# Git Cheat Sheet

Este archivo es un ejemplo de cómo usar Markdown para documentar comandos de Git.

## Comandos útiles

- `git status` – ver estado del repositorio
- `git add .` – agregar todos los cambios al área de preparación
- `git commit -m "mensaje"` – confirmar los cambios
- `git push` – subir los cambios al remoto

## Tips

- Usa `git diff` para ver cambios sin confirmar
- Usa `git log --oneline --graph` para ver el historial de commits de forma compacta
